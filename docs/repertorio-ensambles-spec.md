# Especificación: Programación de Repertorio por Ensamble

## Objetivo
Permitir a los coordinadores de ensamble planificar ciclos anuales mediante propuestas de programas que pueden ser vinculadas o convertidas en programas (giras) reales.

## Modelo de Datos (Supabase)
1. **ciclos_ensambles_anios**:
   - `id`: int8 (PK)
   - `id_ensamble`: int8 (FK -> ensambles.id)
   - `anio`: int4
2. **ciclos_propuestas**:
   - `id`: int8 (PK)
   - `id_ciclo`: int8 (FK -> ciclos_ensambles_anios.id)
   - `nombre_programa`: text
   - `ventana_fechas`: text (Rich Text / Multiline)
   - `observaciones`: text (Rich Text / Multiline)
   - `id_programa`: int8 (FK -> programas.id, nullable)

## Componentes UI
- **Pestaña "Programación de Repertorio"**: Selector de año y acordeones por ciclo.
- **Editor de Propuesta**: Formulario con inputs multilínea para fechas y observaciones.
- **Acción "Asociar/Crear"**: 
  - Si `id_programa` es null: Selector de programas existentes o botón "Crear Programa" que abre `GiraForm`.
- **Vista de Ejecución**: Renderizado de `GiraCard` para propuestas ya vinculadas.

## SQL para Supabase

```sql
-- Crear tabla de ciclos anuales
CREATE TABLE IF NOT EXISTS ciclos_ensambles_anios (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    id_ensamble INT8 REFERENCES ensambles(id) ON DELETE CASCADE,
    anio INT4 NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(id_ensamble, anio)
);

-- Crear tabla de propuestas
CREATE TABLE IF NOT EXISTS ciclos_propuestas (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    id_ciclo INT8 REFERENCES ciclos_ensambles_anios(id) ON DELETE CASCADE,
    nombre_programa TEXT NOT NULL,
    ventana_fechas TEXT,
    observaciones TEXT,
    id_programa INT8 REFERENCES programas(id) ON DELETE SET NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Habilitar RLS (Asumiendo políticas similares al resto del proyecto)
ALTER TABLE ciclos_ensambles_anios ENABLE ROW LEVEL SECURITY;
ALTER TABLE ciclos_propuestas ENABLE ROW LEVEL SECURITY;

-- Nota: Asegúrate de otorgar permisos a los roles autenticados.
```

## Implementación Finalizada

- **Hook de React Query `repertoireCycles`**: 
  - Carga los ciclos anuales (`ciclos_ensambles_anios`) filtrados por `id_ensamble` activo y `anio` seleccionado.
  - Carga las propuestas asociadas (`ciclos_propuestas`) y las agrupa por ciclo.
  - Mapea los programas relevantes a partir de `programasOptions` para mostrar etiquetas amigables.
- **Submódulo UI `RepertoireCyclesTab` en `EnsembleCoordinatorView`**:
  - Selector de año (`repertoireYear`) con rango relativo al año actual.
  - Gestión de ciclos por ensamble activo: creación de ciclo por ensamble y alta de propuestas por ciclo.
  - Edición inline de campos `nombre_programa`, `ventana_fechas` y `observaciones` con persistencia en Supabase.
  - Acción de "Asociar/Crear":
    - Asociación directa de un programa existente mediante `SearchableSelect` (actualiza `id_programa` en `ciclos_propuestas`).
    - Creación de un nuevo programa de tipo `Ensamble` abriendo `GiraForm` pre-rellenado con el nombre de la propuesta y el contexto del ensamble.
    - Vinculación automática de la propuesta recién creada al nuevo programa (`id_programa`) al guardar desde `GiraForm`.
- **Ajuste de `GiraCard`**:
  - Nueva prop opcional `defaultOpenSection`, que al recibir `"repertoire"` fuerza la visibilidad inmediata del `RepertoireManager` al renderizar la tarjeta.
